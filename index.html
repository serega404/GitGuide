Гайд по использованию Gitea сервера

  1. Для начала нужно зарегистрироваться в ZTC Id/RDC Auth/SKBKIT Id, для этого нужно написать администратору и отправить свой логин (который хотите), почту и имя фамилию.

  2. Далее администратор сообщит логин и пароль, после чего вы сможете зайти на сайт https://git.<домен>.ru и зарегистрироваться там, используя свой логин и почту.

Затем выбираем своего провайдера аунтефикации

Вводим свой логин и пароль. После чего вам будет предложено сменить пароль на свой. (Пароли хорошо защищены, посмотреть после смены их никто не сможет, только сбросить на новый)

Настройка клиента

Так как не все Git клиенты поддерживают аунтефикацию через SSO, то для работы с ними нужно использовать токены доступа. Это делается для того, чтобы не передавать свой логин и пароль в клиенте, а использовать токен, который можно легко отозвать в случае компрометации.

Официальный консольный клиент Git поддерживает аунтефикацию через SSO, поэтому для него токены не нужны.

Для использования клиентов Git, таких как Github Desktop, GitKraken и других, необходимо использовать токены доступа. Для этого нужно зайти в настройки своего профиля (в правом верхнем углу) и выбрать раздел "Приложения".

  1. Переходим в раздел Приложения
  2. Вводим название токена (например, "Github desktop")
  3. Раскрываем меню Выбрать разрешения
  4. Выбираем разрешения, которые нужны (для отправки коммитов ставим "repository:Чтение и запись")
  5. Нажимаем Генерировать токен

Теперь сохраняем токен в безопасном месте, так как он будет показан только один раз. Если вы его потеряете, то придется создавать новый.

Пример использования

Для примера будет использовать GitHub Desktop (так как он беслатный и с открытым кодом), но в других клиентах все аналогично.

При первом запуске вам будет предложено войти в свой Github аккаунт, этот шаг по желанию можно пропустить.

Далее необходимо указать Имя Фамилию и почту для подписи коммитов.

Жмём Finish и видим главный экран приложения, так как скорее всего репозиторий уже существует, нужно склонировать его. Для этого нажимаем File -> Clone repository или Ctrl + Shift + O (или Cmd + Shift + O на MacOS).

Теперь выбираем вкладку URL и вставляем ссылку на репозиторий, который вы хотите склонировать. Ссылка должна выглядеть примерно так: https://git.<домен>.ru/<владелец>/<ваш репозиторий>

После этого нажимаем Clone и вводим свои данные для аунтефикации. Логин будет ваш логин, а пароль = токен, который вы создали ранее.

Теперь вы сможете работать с ним как обычно.

Только не забывайте, нажимать кнопку Fetch origin что бы не потерять изменения других участников репозитория.
